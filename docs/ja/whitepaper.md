# ミッション優先のMars Reference Gridホワイトペーパー

**Version:** ミッション優先・採用ドラフト

## Executive Summary
本ホワイトペーパーは、SpaceXのミッション加速を目的としたMars Reference Gridの採用提案です。中核価値は運用面にあり、位置参照の標準化によって調整コストを削減し、着陸候補地の比較と外部解析の検証を迅速化します。Layer 1は単独で成立し、トークンや市場は不要です。NFTは必要に応じた検証・参加の補助に限定され、目的ではありません。

過去の月面土地販売は、信頼できる運用アンカーがなく、所有権の誤読を招いたため失敗しました。本提案は土地所有や主権を否定し、純粋に技術標準としての参照グリッドを提案します。

## 1. Problem: Fragmented Location References Slow Mars Work
火星データは多様な座標系と命名規則に分散しています。統一されたReference Gridがないと、解析結果の比較や外部貢献の検証に時間がかかります。

## 2. Mission-First Goal
ミッション優先の目的は、共通の参照言語によって意思決定を早め、着陸候補地の検証と共有を容易にすることです。

## 3. Misinterpretation Guardrails
- **土地所有ではない** — 所有権や主権の主張はない。
- **市場ではない** — 売買や取引は必須ではない。
- **NFTのためのミッションではない** — ミッションはトークンと無関係。

## 4. Architecture Overview (3-Layer Model)
### Layer 1 (Must-have): Official Mars Reference Grid / Reference IDs
Layer 1はReference GridのスキーマとReference IDsを定義する最小構成です。単独で成立し、所有権や市場の概念を含みません。

### Layer 2 (Optional core): Contribution & Verification
Layer 2は外部貢献の受付と検証を担います。独立アルゴリズムや市民科学の成果を再現性と複数ソース検証で評価し、SpaceXは必要な成果のみ採用できます。

### Layer 3 (Optional only): Monetization (Appendix)
Layer 3は付録の任意要素であり、ミッション価値とは切り離します。

## 5. Layer 1 Specification (Reference Grid)
Reference Gridは以下を定義します。
- 投影法とエポック。
- 解像度階層とIDフォーマット。
- 出典、時刻、バージョンなどのメタデータ。

## 6. Layer 2 Specification (Contribution & Verification)
貢献レイヤーは以下を含みます。
- 提出要件（データ、手法、ツール、出典）。
- 再現性チェック。
- 競合解析の解決プロセス。

## 7. Non-Financial Benefits to SpaceX
- 着陸候補地の比較が迅速化。
- 内部ドキュメントの曖昧性が低下。
- 対外説明の位置参照が明確化。
- 外部データ統合の信頼性向上。

## 8. Implementation Approach (Phase 0–2)
- **Phase 0: Reference-only stop point** — 参照グリッドを試行し、価値が不明確なら停止。
- **Phase 1: Contribution & Verification** — 外部提出を受け、再現性で検証。
- **Phase 2: Scaled Integration** — 検証済みデータを運用・広報に統合。

## 9. Why SpaceX Is the Right Anchor
SpaceXは火星運用の現実的アンカーです。SpaceXが参照標準を認めれば、研究者やツール開発者はそれに合わせられます。所有権や商用目的の主張は含みません。

## Appendix: Optional Monetization (Non-essential)
SpaceXが求める場合のみ、任意の資金化オプションを検討できます。これらは**未検証の仮定**として明示され、ミッション判断には影響しません。
