# ミッション優先のSpaceX採用提案：Mars Reference Grid

## Executive Hook — Why SpaceX Adopts
SpaceXは火星ミッションの加速を目的に、位置参照の標準化によって調整コストを削減し、着陸候補地の比較と意思決定を速めることができます。本提案はミッション優先であり、市場や収益を前提としません。

本提案はモジュール構成です。Layer 1は単独で価値が成立し、Layer 2とLayer 3は任意です。NFTは目的ではなく、必要に応じた検証・参加の補助に限定されます。

## Misinterpretation Guardrails
- これは**土地所有・不動産・主権**ではありません。
- SpaceXは**NFTのためにロケットを打ち上げません**。
- Reference Gridは**技術標準**であり、所有権の台帳ではありません。
- 収益化は**付録のみ**で、採用条件ではありません。

## Why SpaceX (Strategic Benefits)
- **ミッション加速**：候補地評価の比較・検証を迅速化。
- **コスト削減**：参照のズレによる調整コストを低減。
- **検証済み候補地**：外部解析の採用判断が容易。
- **エコシステム衛生**：科学参照と所有権の分離を徹底。

## 3-Layer Modular Adoption
**Layer 1 (Must-have): Official Mars Reference Grid / Reference IDs**
- 参照IDによる座標標準。単独で成立。
- トークンや市場、所有権は不要。

**Layer 2 (Optional core): Contribution & Verification**
- 独立アルゴリズムや市民科学の貢献を受け入れ。
- 再現性と複数ソース検証を重視。

**Layer 3 (Optional only): Monetization**
- 付録に隔離し、必須ではない。
- 研究・運用価値が主で、収益は副次。

## Minimal SpaceX Support Required
- Reference IDスキーマのパイロットまたは公開の承認（Phase 0）。
- 最小限の技術窓口。
- 公開可能な参照データへのアクセス（任意）。

## Implementation Snapshot (Phase 0–2)
- **Phase 0: Reference-only stop point** — 参照グリッドを導入し、曖昧性低減がなければ停止。
- **Phase 1: Contribution & Verification** — 外部提出を受け、検証済みのみ採用。
- **Phase 2: Scaled Integration** — 有用なデータを運用・広報に統合。

## Expected Outcomes / KPI Candidates (Non-financial)
- 座標参照の曖昧性低下。
- 外部協力の立ち上げ時間短縮。
- 検証済み候補地・データの増加。
- 対外説明における補足や訂正の減少。

## Commercial Terms as Options (Neutral)
商業条件は任意かつ非拘束であり、SpaceXが求める場合のみ検討します。ミッション価値を損なわない中立的な扱いとします。
